# CEREBRO

**Cross-hardware EEG Representation Encoding with Biometric Response Optimization**

A TRIBE-style multi-subject transformer for visual EEG encoding on THINGS-EEG2, with cross-hardware transfer to consumer-grade Alljoined-1.6M EEG.

**[Demo](https://b-a-1-a.github.io/CEREBRO/) · [Report](report/writeup.pdf)**

> **Note:** The repo is a bit messy right now — training was run in Colab and artifacts live on Google Drive. A cleaned-up, fully self-contained version is coming soon.

---

## What this is

We test whether the central trick of [TRIBE v2](https://ai.meta.com/research/publications/a-foundation-model-of-vision-audition-and-language-for-in-silico-neuroscience/) — a single transformer trained jointly across all subjects with per-subject readout blocks — transfers from fMRI to event-related EEG. We run it on [THINGS-EEG2](https://www.sciencedirect.com/science/article/pii/S1053811922007947) (10 subjects, 64-ch BioSemi, 100 Hz) and then transfer the frozen encoder to [Alljoined-1.6M](https://arxiv.org/abs/2508.18571) (32-ch Emotiv Flex 2, consumer-grade).

Headline results:
- Multi-subject CLIP transformer: **R = 0.091 ± 0.015** vs linear baseline 0.042 (paired-t p << 0.001, N=10 subjects, 3-seed avg)
- 32-channel intersection model (used for transfer): **R = 0.122 ± 0.019**
- Cross-hardware transfer, Variant B (frozen encoder + fresh subject head): **posterior R = 0.030 ± 0.015** vs scratch ridge 0.007 (p = 0.034, N=5 Alljoined subjects)

## Repo layout

```
CEREBRO/
  src/                  # model definitions, data loaders, training loop, eval
  scripts/              # one-off scripts: extract embeddings, run experiment, collect report artifacts
  configs/              # YAML configs for baseline, per-subject, and multi-subject runs
  report/               # Quarto writeup (writeup.qmd + report_metrics.py + references.bib)
  figures/              # result figures (generated by Phase 6 notebook cells)
  results/              # CSVs: all_results.csv (Phase 5), alljoined_transfer.csv (Phase 7)
  embeddings/           # cached CLIP and DINOv2 embeddings (.npy)
  logs/                 # STATE.json, channel_intersection.json, phase logs
  notebook.ipynb        # full experiment notebook (run in Colab; see below)
```

The actual training was done in a [Google Colab notebook](https://colab.research.google.com/drive/1Le9zgK1qOpROdRqXrA9zBA0LpOSM7ZRU) with artifacts saved to Google Drive at `/content/drive/MyDrive/tribe-eeg/`. The files in `figures/`, `results/`, `embeddings/`, and `logs/` here are the exported artifacts from that Drive folder.

## Setup

```bash
git clone https://github.com/B-a-1-a/CEREBRO.git
cd CEREBRO
pip install -r requirements.txt
```

Python 3.10+ recommended. PyTorch with CUDA is needed for training; CPU-only is fine for eval and the report render.

### Data

The experiment uses two datasets that are not included in this repo:

- **THINGS-EEG2** preprocessed EEG: download from OSF ([`anp5v`](https://osf.io/anp5v/)) — the outer zip is ~51 GB. Use the 63-channel variant (`sub-XX__63_channels.zip`).
- **THINGS-EEG2 images**: download from OSF ([`y63gw`](https://osf.io/y63gw/)) — fetch files individually via the OSF API (the `?zip=` bundle endpoint is broken for this resource).
- **Alljoined-1.6M**: download from HuggingFace.

Set up the directory tree under a persistent root (e.g., Google Drive):
```
tribe-eeg/
  raw/thingseeg2_preproc/sub-{01..10}/
  raw/thingseeg2_metadata/{training_images.zip, test_images.zip}
  processed/
  embeddings/
  checkpoints/
  results/
  logs/
  figures/
```

### Reproducing the experiment

The full pipeline is in `notebook.ipynb` (designed for Colab with Drive mounted). Each phase has its own section:

| Phase | What it does |
|---|---|
| 0 | Environment setup, dir tree, STATE.json |
| 1 | Download and stream-extract EEG + images |
| 2 | Average EEG reps, build image manifests |
| 3 | Extract CLIP ViT-L/14 and DINOv2-Large embeddings |
| 4 | Ridge sanity baseline (Pearson R gate) |
| 5 | Train linear, per-subject, and multi-subject transformer models |
| 6 | Evaluation figures and all_results.csv |
| 7 | Cross-hardware transfer to Alljoined-1.6M |
| 8 | Report |

For standalone training (if you have the embeddings and averaged EEG already):
```bash
python scripts/run_experiment.py --config configs/transformer_multi_subject.yaml
```

### Rendering the report

Requires [Quarto](https://quarto.org/docs/get-started/) and the artifact files in `figures/`, `results/`, `embeddings/`, and `logs/`.

```bash
quarto render report/writeup.qmd --to pdf
```

To collect only the artifacts needed for the report from a full Drive export:
```bash
python scripts/collect_report_artifacts.py --source /path/to/tribe-eeg --dest .
```
