     TRIBE-style EEG encoding on THINGS-EEG2 — execution plan

     Context

     The user is testing whether TRIBE v2's central trick — a
     subject-conditioned transformer trained jointly across all participants —
     transfers from slow fMRI (TRIBE's home turf) to fast event-related EEG. The
      hypothesis: a multi-subject transformer with a per-subject readout block
     beats per-subject-only baselines on THINGS-EEG2, and the recipe transfers
     to consumer-grade Emotiv hardware (Alljoined-1.6M).

     The user has provided an exhaustive 8-phase spec. This plan is the
     execution map, not a re-design — it captures how phases hand off to each
     other, where checkpoints live, and what design decisions are baked in. The
     spec text is authoritative; this file is the index.

     The main novel architectural piece (vs. the TRIBE paper) is the RSVP 
     context window: each predicted EEG epoch sees the 10 surrounding RSVP
     trials' image embeddings as transformer context. That's the part of the
     methodology section that needs the most care to document.

     Operating environment

     - Compute: Colab session, ideally H100 (target ~5 H100-hours total). Falls
     back to T4/A100 with proportionally longer wall-clocks.
     - Notebook: existing CEREBRO Colab notebook (URL in memory). One
     mega-notebook, sections marked with markdown headers per phase. Currently
     empty (one stub cell + "hello world").
     - Persistence: everything to /content/drive/MyDrive/tribe-eeg/. Working
     scratch on /content/work/. Local laptop sees nothing.
     - Pacing: print-and-proceed for long ops (download size + wall-clock
     estimate + cell preview, then run). User interrupts only if objecting. No
     chat-level "go" handshake.
     - Source of truth for the spec: the user's prompt above. This plan defers
     to it on parameters, sanity criteria, and architecture details.

     Phase map

     Each phase ends with concrete artifacts on Drive that gate the next phase.
     Skip-if-exists logic on every step (force=False flag).

     Phase 0 — Environment setup

     - Output: directory tree under /content/drive/MyDrive/tribe-eeg/,
     logs/env.json with GPU info, package versions printed.
     - Gate to next: nvidia-smi shows a GPU (warn if not H100); Drive mounted;
     subdirs exist.

     Phase 1 — Data acquisition (one-time, ~30 min)

     - 1a: Preprocessed THINGS-EEG2 EEG from OSF (anp5v) →
     raw/thingseeg2_preproc/sub-{01..10}/. Download to /content, unzip
     per-subject inner zips, move to Drive. Skip if all 10 subject dirs present.
     - 1b: Image metadata from OSF (y63gw) →
     raw/thingseeg2_metadata/{training_images,test_images}/. 1654 training
     concepts × 10 imgs + 200 test concepts × 1 img.
     - 1c: Inventory — assert counts, save logs/data_inventory.json.

     Phase 2 — EEG averaging (CPU, ~10 min)

     - Inspect first: open sub-01's actual file layout before writing the loop.
     Spec assumes (16540, 4, C, T) train and (200, 80, C, T) test at 100 Hz.
     Surface deviations to user before averaging.
     - Average across reps → processed/eeg_{train,test}_avg_sub-{01..10}.npz.
     np.savez_compressed with key eeg.
     - Image filename manifests: walk training_images/test_images in
     alphabetical concept-then-image order (canonical THINGS-EEG2 ordering),
     save processed/image_filenames_{train,test}.json. Assert lengths 16540
     / 200.
     - Sanity figure: figures/sanity_evoked.png — mean ERP across subjects/test
     images/channels, expect P1/N170 around 100–200 ms.

     Phase 3 — Image embeddings (~20 min on H100)

     - CLIP ViT-L/14 → embeddings/clip_vitl14_{train,test}.npy (16540×768,
     200×768).
     - DINOv2-Large CLS token → embeddings/dinov2_large_{train,test}.npy
     (16540×1024, 200×1024).
     - Skip-if-exists per file. Sanity: no NaNs/zero rows; same-concept cosine
     similarity > random pair.

     Phase 4 — Ridge sanity baseline (CPU, ~5 min)

     - Per-subject sklearn Ridge from CLIP → flattened EEG. Pearson R per
     (channel, timepoint).
     - Gate: mean R across subjects in [0.10, 0.25]. <0.05 → alignment broken
     (almost always image filename ordering); >0.4 → leak. Stop and debug if
     either.
     - Output: results/phase4_ridge_baseline.json.

     Phase 5 — Three models (~2 H100-hr core + ~1 H100-hr seed averaging)

     Three runs, all evaluated on the 200-image test set:

     1. Linear baseline (PyTorch for fairness): Linear(D_emb → C·T), per
     subject. → checkpoints/linear_baseline/sub-{01..10}.pt.
     2. Per-subject transformer: same encoder as #3 but final head is
     Linear(d_model → C·T), no subject block. Trained per subject. →
     checkpoints/per_subject_transformer/sub-{01..10}.pt.
     3. Multi-subject transformer: full architecture per spec. Trained jointly
     on all 10 subjects. → checkpoints/multi_subject_transformer/best.pt.

     Architecture spec (from the user prompt — anchored here for execution):
     - Stem: Linear(D_emb → 384) + LayerNorm.
     - Encoder: 4 layers, 6 heads, d_model=384, ffn=1536, learned positional
     embeddings over (1+K=11) tokens.
     - Subject block: (n_subjects, 384, C·T) parameter, indexed by subject_id.
     Read out from position-0 token.
     - Loss: MSE on (B, C, T).

     Training config: AdamW lr=1e-4, wd=0.01, cosine + 5% warmup, batch 64, 30
     epochs, early stop patience 5 on val Pearson R. Concept-level 90/10
     train/val split (no concept appears in both).

     Two design choices to bake in now (not in spec body, but required by Phase 
     7):
     - Subject dropout during multi-subject training — randomly mask the
     subject_id with p≈0.1, route those samples through a learned "unseen
     subject" pathway (averaged or zero-init subject embedding). This is what
     makes Phase 7 Variant A (zero-shot transfer) possible without retraining.
     The spec calls this out in §7c; I'm hoisting it to Phase 5 here.
     - RSVP context window source: use the natural sequential array index as
     presentation order within each session, pad zeros at session boundaries.
     Document explicitly in logs/context_source.json so the methods section can
     cite it.

     Seed averaging: re-run multi-subject (CLIP) with seeds 1337 and 2026 (in
     addition to 42). Average results in Phase 6.

     DINOv2 ablation: re-run multi-subject only with DINOv2 embeddings (single
     seed 42). → checkpoints/multi_subject_transformer/best_dinov2.pt.

     Channel-subset variant for Phase 7: after the 17-channel main run lands,
     retrain multi-subject (CLIP) with the output head restricted to the
     THINGS-EEG2 ↔ Alljoined channel intersection (computed in 7b). →
     checkpoints/multi_subject_transformer/best_channelsubset.pt. Budgeted in
     the H100 plan; runs back-to-back with main Phase 5.

     Sanity gate: test Pearson R(multi-subject) > R(per-subject) > R(linear). If
      violated, stop.

     Phase 6 — Eval and figures

     - figures/headline_boxplot.{png,pdf} — three columns × 10 dots, paired-line
      connectors, FDR-corrected paired t-tests.
     - figures/channel_time_heatmap.png — multi-subject CLIP, channel ×
     timepoint Pearson R, channels sorted occipital→frontal.
     - figures/embedding_ablation.png — CLIP vs DINOv2, multi-subject only.
     - results/all_results.csv — long-form: model, embedding, subject, mean_r,
     max_r.

     Phase 7 — Alljoined cross-hardware transfer (~2 H100-hr)

     - 7a: download Alljoined-1.6M from HuggingFace. Pause first: surface
     candidate dataset cards, user picks URL.
     - 7b: channel intersection (THINGS-EEG2 17ch BioSemi ∩ Alljoined 32ch
     Emotiv via 10-20 labels). Use
     checkpoints/multi_subject_transformer/best_channelsubset.pt as the
     transferable model.
     - 7c: three transfer variants:
       - A — zero-shot: route Alljoined subjects through the unseen-subject
     pathway baked in during Phase 5. → results/alljoined_zero_shot.json.
       - B — subject-block-only fine-tune: freeze backbone, init new subject
     block with rank-128 low-rank, train 1 epoch on half each subject's data. →
     results/alljoined_subject_block_ft.json.
       - C — full fine-tune low LR: unfreeze, lr=1e-5, 1 epoch. →
     results/alljoined_full_ft.json.
     - 7d: per-subject ridge from CLIP → Alljoined EEG, scratch baseline.
     - 7e: figures/alljoined_transfer.png — bar chart of [scratch ridge, A, B,
     C] with per-Alljoined-subject error bars.

     Phase 8 — Report

     Five-page report following syllabus rubric (intro, related work, methods,
     results, discussion). Pulls all figures from figures/ and numbers from
     results/. Outline in spec §Phase 8.
